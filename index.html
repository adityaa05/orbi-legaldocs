Privacy Policy

Orbi was designed with privacy as a first principle, not an afterthought.

Last updated — April 13, 2026
The short version

Orbi collects nothing about you. No accounts, no analytics, no message content logging — ever. Your messages are end-to-end encrypted and relayed through a blind server that cannot read them. Signal metadata (action type, sender name) is transient and never persisted.

What we collect

Orbi does not collect personal information in any traditional sense. There are no user accounts, no sign-ups, no email addresses, and no persistent identifiers.

The following minimal data is handled — and only as described below:

  • Display name — entered by you, stored only in your browser's local storage. Never sent to our servers.
  • Session code — the 6-digit code you generate or enter, stored locally for auto-reconnect. The server only sees this as a room identifier, never associated with an identity.
  • Signal payloads — encrypted on your device before transmission. The relay server forwards them without reading or storing them.
  • IP address — temporarily held in server process memory for rate-limiting only (max 90 seconds). Never written to disk or logged.
  • Tab screenshots — only when you explicitly trigger "Send tab via Orbi". Encrypted before transmission and never stored by the server.

End-to-end encryption

Message payloads are encrypted on your device using AES-256-GCM before transmission. The encryption key is derived from the 6-digit session code and a per-session server nonce using PBKDF2 (100,000 iterations, SHA-256). The server never possesses the key and cannot read payload contents.

Note that signal metadata — the action type (e.g. flash, ping) and sender display name — travels unencrypted so the relay can route it. This metadata is never written to disk or logged.

Transient signaling server

Orbi's server acts as a blind relay — it routes encrypted payloads between peers and immediately discards them. No message contents are written to any database or log file.

To prevent abuse, the server temporarily holds your IP address in memory for rate-limiting purposes (max 60 joins per minute). This data exists only in process memory, is never written to disk, and is evicted after 90 seconds of inactivity.

Rooms are automatically deleted from server memory after 5 minutes of inactivity.

Zero tracking

Orbi contains no analytics SDK, no telemetry, no cookies, and no third-party tracking of any kind. We don't know how many people use Orbi, how often they use it, or what they say.

Local storage only

Orbi uses Chrome's chrome.storage.local to persist the following between sessions: your display name, volume preference, the last active session code (for auto-reconnect), presentation mode state, and a debug flag. This data never leaves your device.

Third-party services

Orbi does not integrate with any third-party services. No Google, Facebook, Mixpanel, Sentry, or equivalent SDK is included in the extension or server.

The extension's only outbound network connection is the WebSocket to Orbi's signaling server. No data is sent anywhere else.

Tab sharing (optional)

Orbi includes an optional right-click context menu item — "Send tab via Orbi" — that captures a JPEG screenshot of the visible tab and sends it to your connected peer as an encrypted payload over the existing WebSocket.

This feature is only triggered by explicit user action. No tab content is captured passively. The screenshot is encrypted on your device before transmission and is never stored or logged by the server.

Open source

Orbi's source code is publicly available for independent review at github.com/adityaa05. Our privacy claims are verifiable — not just stated.

Changes to this policy

If this policy ever changes in a material way, the "Last Updated" date at the top will reflect that and we will update the policy at this URL. We will never reduce your privacy rights without clear notice.
